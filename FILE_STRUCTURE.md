## File Purposes

### Core Extension Files
- **manifest.json**: Defines extension metadata, permissions, entry points
- **background.ts**: Runs continuously, handles initialization and message routing
- **content.ts**: Injected into pages, detects forms and fills them
- **popup.tsx + components**: User interface for profile management

### Services (Business Logic - Framework Independent)
- **storageService.ts**: Abstraction over Chrome Storage API
- **profileService.ts**: Profile lifecycle management
- **formDetectionService.ts**: Field pattern recognition
- **autofillService.ts**: DOM manipulation and field filling

### UI Components (React)
- **Popup.tsx**: State orchestration and view routing
- **ProfileList.tsx**: Lists all profiles, allows selection
- **ProfileEditor.tsx**: Form for creating/editing profiles
- **MasterPasswordCheck.tsx**: Password verification modal
- **AutofillButton.tsx**: Autofill trigger with status feedback

### Types & Interfaces
- **types/index.ts**: All TypeScript interfaces for type safety

### Configuration
- **package.json**: npm dependencies and scripts
- **tsconfig.json**: TypeScript compiler options
- **webpack.config.js**: Webpack bundler configuration

### Documentation
- **README.md**: Complete project overview and usage
- **QUICK_START.md**: Fast setup guide (5 minutes)
- **DEVELOPMENT.md**: Developer guide and best practices
- **ARCHITECTURE.md**: Design decisions and technical details
- **CHANGELOG.md**: Version history and planned features

### Build & Deployment
- **build.sh / build.bat**: One-command build scripts
- **.gitignore**: Git configuration

## Data Flow

```
User Creates Profile
    ↓
Popup.tsx captures input
    ↓
ProfileEditor.tsx validates
    ↓
Background stores via StorageService
    ↓
Data saved to Chrome Storage

---

User Visits Form Page
    ↓
Content.ts detects form fields
    ↓
FormDetectionService patterns match
    ↓
Popup shows detected fields count

---

User Clicks Autofill
    ↓
Master password verification
    ↓
AutofillService gets profile data
    ↓
Content.ts fills form fields
    ↓
User sees populated form ✓
```

## Message Communication

```
Popup  ←→  Background Service Worker
 ↓             ↓
 │  runtime.sendMessage()
 │  (GET_PROFILES, SAVE_PROFILE, etc.)
 │
 └──→ Content Script via tabs.sendMessage()
      (DETECT_FIELDS, AUTOFILL_FORM)
      ↓
      Page interaction
      Form detection & filling
```

## Extensions Points

### To Add a New Form Field Type:
1. Add pattern to `formDetectionService.ts`
2. Add case to `autofillService.ts`
3. Add interface field to `types/index.ts`
4. Add input to `ProfileEditor.tsx`

### To Add a New Profile Type:
1. Update `Profile.type` union in `types/index.ts`
2. Add default in `profileService.ts`
3. Update dropdown in `ProfileEditor.tsx`

### To Add New Functionality:
1. Add service method if it involves data/detection
2. Add component if it involves UI
3. Add message handler in `background.ts`
4. Update types in `types/index.ts`

## Load into Chrome

1. `npm install`
2. `npm run build`
3. Go to `chrome://extensions/`
4. Enable Developer Mode (top right)
5. Click "Load unpacked"
6. Select the `dist/` folder
7. Done! 🎉
